## 1. 사용자 초기 설계 7단계 (원점)

사용자 원문:
> 원본 파일 먼저 열어서 CMYK와 요소 확인 → 새 파일 생성 → 패턴선 복사 후 패치 → 원본파일 베인 색상 패턴선 안 채우기 → 원본 파일 요소 분석해서 각 사이즈별 면적비에 맞게 스케일링 → 새 파일의 각 패턴 조각 위에 위치 찾아서 배치 → EPS 저장

### 단계 정리

| # | 단계 | 동작 | 현재 코드 대응 |
|---|------|------|---------------|
| 1 | 원본 파일 열기 | 디자인 AI 파일 open + CMYK 문서 확인 | STEP 1~2 |
| 2 | 새 파일 생성 | 빈 CMYK 베이스 문서 (SVG 아트보드 크기) | STEP 5 후반 |
| 3 | 패턴선 복사+패치 | SVG path → 베이스 문서로 duplicate | STEP 7 `importSvgPathsToDoc` |
| 4 | 색 채워진 패턴선 안 채우기 | 원본 몸판 색으로 **채우지 않은** 장식선은 건드리지 않음 | (해석 필요 — 현재 Phase 1 filled 체크와 다를 수 있음) |
| 5 | 면적비 스케일링 | `linearScale = sqrt(targetArea / baseArea)` | STEP 9 |
| 6 | 각 패턴 조각 위 배치 | 요소를 조각 위에 위치 찾아 배치 | STEP 10 (현재 D1/D2로 복잡) |
| 7 | EPS 저장 | saveAs EPS/PDF | STEP 11-C |

### "해석 과제" — 사용자 확인 필요 (Phase A)

| # | 질문 | 옵션 |
|---|------|------|
| I1 | "패턴선 복사 후 패치"의 "패치"란? | (a) 베이스 문서에 이식/삽입 (b) 색 채우기 (c) 수정/덮기 |
| I2 | "색상이 칠해진 패턴선은 안 채우기"의 의미? | (a) 원본 몸판에서 이미 색 있는 path는 **재덮지 않음** (b) 장식선(fill=없음)은 색 채우기 대상 아님 (현재 Phase 1 동작) (c) 둘 다 |
| I3 | "면적비에 맞게 스케일링" 대상은? | (a) 요소만 (현재) (b) 전체 (c) 패턴선만 |
| I4 | "각 패턴 조각 위에 위치 찾아서 배치" 정밀도? | (a) 조각별 독립 배치 (현재 D2) (b) 몸판 전체 중심 1회 (구 D1) (c) 원본 위치 그대로 보존 |
| I5 | EPS or PDF? | (a) EPS 기본 (b) PDF 기본 (c) config에서 선택 (현재) |

## 3. 재구축 접근법 3가지 비교

### 접근 A — 특정 커밋 롤백 + 최소 픽스
- 방법: `git revert` 혹은 `git restore` 로 dd51cc5 또는 1985909 시점 jsx 복원
- 그 후 버그 B(duplicate) 핀포인트 패치만 재적용
- **장점**: 2~3시간, 검증된 시점
- **단점**: 그 후 개선분 일부 누락 (auto-color/교체용요소/Phase 2)
- **위험**: git 작업 중 충돌 가능

### 접근 B — 빈 파일에서 새로 작성 ★권장
- 방법: `grading-v2.jsx` 신규 파일, 사용자 7단계 흐름 그대로 500~800줄 목표
- 검증된 로직(면적 공식, duplicate 패턴, CMYK 문서 생성)만 dd51cc5에서 발췌 인용
- **장점**: 가장 깨끗, "원점" 요청 부합, 잔재 0
- **단점**: 4~6시간
- **위험**: 기존 미세 픽스 재발견 필요 (버그 B duplicate 전환 같은 건 처음부터 반영)

### 접근 C — 현재 코드에서 "정원 가꾸기"
- 방법: 현재 2128줄에서 Phase 2/D1/D2/교체용요소 잔재/APCA/방어 로직 제거
- **장점**: 점진적, 2~3시간
- **단점**: 제거 누락 가능성, 여전히 1000~1500줄 예상
- **위험**: "깨끗한 원점"과는 거리 있음

## 4. 권장안: 접근 B (새로 작성)

### 근거
1. 사용자 "원점에서" 명시 — B가 가장 부합
2. "불필요한 코드와 규칙도 모두 제거" 요청 — B가 최대 달성
3. 논리 오류 재발 위험 최소 — 새 코드라 잔재 없음
4. 참조 베이스 dd51cc5 존재 → 백지에서 시작하지 않음, 검증된 구조 가져옴
5. v1 보존 → 실패 시 즉시 롤백

### 차선 (사용자가 시간 제약 원하면): 접근 A
- `git checkout dd51cc5 -- illustrator-scripts/grading.jsx`
- + 버그 B duplicate 패치만 재적용
- 2~3시간에 실측 가능

## 5. Phase B~D 구현 계획 (접근 B 기준)

### Phase A (30분) — 사용자 초기 설계 확인 ★의사결정
- 위 §1 "해석 과제" I1~I5 사용자와 확정
- 접근법 선택 (B / A / C)
- 아래 §9 의사결정 5개 확정

### Phase B (1~2시간) — 뼈대 작성
산출물: `illustrator-scripts/grading-v2.jsx` (신규, 500~800줄 목표)

구조:
```
STEP 0: config.json 읽기
STEP 1: 디자인 AI 파일 열기 (CMYK 검증)
STEP 2: baseArea 측정 + 메인 색상 추출
STEP 3: 요소 수집 (duplicate 대기)
STEP 4: 패턴 SVG 열기 + 빈 CMYK 베이스 문서 생성
STEP 5: 패턴선 복사 (importSvgPathsToDoc)
STEP 6: 요소 duplicate → 베이스 문서에 paste
STEP 7: 면적비 스케일링 (linearScale)
STEP 8: 위치 배치 (사용자 I4 선택에 따라 단순한 알고리즘 하나만)
STEP 9: 레이어 정리 + 저장 (EPS)
STEP 10: result.json 기록
```

원칙:
- 각 STEP 50~100줄 이내
- `$.writeln` 기본 로그만, 파일 로그(writeLog) 제거
- try/catch 최소 — 메인 try 1개 + catch 1개
- 방어 안전망 없음 (논리가 맞으면 불필요)

### Phase C (2~3시간) — 단계별 실측 검증
- 사이즈 2XS / L / 4XL 3개 테스트
- 각 STEP 실패 시 즉시 로그로 확인
- 색 채움/위치/스케일 육안 확인
- 실패 지점 기록

### Phase D (1시간) — Fine-tuning
- 작은 수정만 허용
- 근본 로직 건드리지 않기
- 3사이즈 모두 통과하면 v2를 v1으로 rename → 기존 grading.jsx 아카이브 (`grading.v1.jsx.bak`)

## 10. 위험 완화

### 롤백 경로
1. v1(`grading.jsx`)는 변경 금지, 그대로 보존
2. v2는 `grading-v2.jsx` 신규 파일
3. Rust 쪽에서 jsx 경로 결정하는 부분 확인 필요 (현재는 `illustrator-scripts/grading.jsx` 하드코딩일 가능성)
   - 옵션 (i): 환경변수 `GRADING_V2=1` 로 선택 (Rust 수정 필요)
   - 옵션 (ii): v2 검증 끝나면 파일 이름 스왑 (Rust 무수정)
4. v2 실패 시: `grading-v2.jsx` 삭제 → v1 원복

### 시간 초과 대비
- Phase B에서 2시간 이상 걸리면 → 접근 A(dd51cc5 롤백)로 전환 검토
- Phase C에서 3사이즈 통과 못하면 → 원본 dd51cc5와 diff 뜨기

### 기능 누락 대비
- Phase A에서 I1~I5 확정 → 사용자 초기 설계 해석 일치 보장
- v2는 최소 핵심만, 부가 기능은 Phase D 이후 하나씩 재도입
