# Changelog

Grader 앱의 버전별 변경사항을 기록합니다.

이 문서는 [Keep a Changelog](https://keepachangelog.com/ko/1.1.0/) 형식을 따르고,
버전 번호는 [유의적 버전(SemVer)](https://semver.org/lang/ko/) 규칙을 따릅니다.

## 버전 번호 읽는 법 (간단 안내)

**형식**: `major.minor.patch` (예: `1.0.0`)

| 자리 | 언제 올라가나 | 예시 |
|-----|-------------|------|
| **major** (첫 번째) | 기존 데이터 형식이 바뀌어 구버전과 호환이 끊길 때 | 1.0.0 → 2.0.0 |
| **minor** (두 번째) | 새 기능 추가 (기존 데이터는 그대로 읽힘) | 1.0.0 → 1.1.0 |
| **patch** (세 번째) | 버그 수정만 | 1.0.0 → 1.0.1 |

**pre-release**: `1.1.0-beta`, `1.1.0-rc.1` 처럼 하이픈 뒤에 이름이 붙으면 **테스트용 임시 버전**. 일반 직원에게는 자동 업데이트로 전파되지 않습니다.

## [1.0.1] — 2026-04-28

🩹 **첫 패치 릴리스.** v1.0.0 배포 후 발견된 표시 결함 + 자동 업데이트 안내문 결함 + 에러 진단성 결함을 정리합니다.

### Fixed (버그 수정)

- **앱 버전 표시가 항상 v0.1.0으로 보이던 표시 버그**
  - 좌하단 상태바와 설정 > 정보 섹션의 버전 문자열이 코드에 직접 박혀 있어, 새 버전을 설치해도 옛 숫자가 그대로 보였습니다.
  - 이제 `package.json`의 버전을 빌드 시점에 자동으로 끌어오도록 바뀌어, 다음 릴리스부터는 항상 정확한 버전이 표시됩니다.
- **자동 업데이트 모달의 안내문이 placeholder("자동 생성된 릴리스입니다…")로 노출되던 결함**
  - v1.0.0 배포 시 `latest.json`의 안내문 필드에 내부 메모가 그대로 들어가, 자동 업데이트를 받는 직원 PC에 이상한 텍스트가 노출될 위험이 있었습니다.
  - 이제 릴리스 워크플로우(`.github/workflows/release.yml`)가 `CHANGELOG.md`의 해당 버전 섹션을 자동으로 추출해 안내문에 넣습니다.
- **그레이딩 실패 시 "알 수 없는 오류"로 가려져 진짜 원인을 못 보던 결함**
  - 사이즈별 catch / 전체 catch 두 곳 모두 `String(e)` fallback을 통과시켜, Illustrator timeout 같은 실제 메시지가 화면에 그대로 표시됩니다.
  - 시트 가져오기(라인 467) / 폴더 열기(라인 648)와 동일한 처리로 통일했습니다.

### Internal (내부 구조 변경, 일반 사용자 영향 없음)

- **버전 단일 출처(SSOT) 정립**: `package.json`의 version 필드가 프론트 표시의 단일 소스. `vite.config.ts`의 `define` 옵션으로 `__APP_VERSION__` 빌드 타임 상수 주입.
- **`vite-env.d.ts`**에 `__APP_VERSION__` 글로벌 타입 선언 추가.
- **`release.yml`**에 "Extract CHANGELOG section" step 추가 — awk 기반 섹션 추출 + GITHUB_OUTPUT heredoc + fallback 메시지.

## [1.0.0] — 2026-04-22

🎉 **첫 공식 릴리스!** 내부 직원 배포용.

이번 버전부터 GitHub Releases를 통한 **자동 업데이트**가 활성화됩니다.
앞으로 앱을 켤 때마다 새 버전이 있는지 자동으로 체크하고 알려드립니다.

### Added (신규 기능)

- **양면 유니폼 그레이딩 지원**
  - U넥 양면유니폼 스탠다드의 앞/뒤 양면을 한 번에 그레이딩
  - 이름 기반 매칭(표1/이1 등 페어 이름) 방식 도입
  - 4분면 색상 구분(표1↔표2, 이1↔이2)으로 패턴 파편 식별 용이
- **SVG 일괄 표준화 기능**
  - 패턴 관리 카드의 `⋮` 더보기 메뉴 → `[📐 SVG 표준화]` 로 실행
  - 시뮬레이션(미리보기) → 실제 변환 → Drive 자동 재스캔까지 한 흐름으로 처리
  - 자동 백업 옵션 제공 (기본값: 켜짐, 권장 유지)
  - ⚠️ **현재 지원 범위**: **U넥 양면유니폼 스탠다드 전용** (`NORMALIZER_VERSION = "1.0-uneck-double-sided"`)
  - 슬림, V넥, 하의 등 다른 분류는 **추후 업데이트 예정**. 해당 폴더에서 표준화 버튼을 눌러도 파일이 수정되지 않고 실패 메시지가 뜹니다.
- **자동 업데이트 시스템**
  - 앱 시작 시 GitHub Releases에서 새 버전 자동 확인
  - "새 버전 있어요" 모달로 안내 + 다운로드 진행률 표시 + 서명 검증 후 자동 재시작
  - 설정 페이지 > "버전 정보" 섹션에서 수동 [업데이트 확인] 버튼도 제공
  - 모든 업데이트는 **선택형** (사용자가 "나중에" 가능)
- **G드라이브 신규 사이즈 SVG 자동 감지**
  - 디자이너가 G드라이브 폴더에 새 사이즈(예: 5XL) SVG를 추가하면 앱이 자동으로 인식
  - 기존 사이즈 치수는 보존한 채 신규 사이즈만 병합 (덮어쓰기 방지)
- **패턴 카드 `⋮` 더보기 메뉴**
  - 즐겨찾기 별 옆에 위치한 더보기 버튼
  - 현재는 [📐 SVG 표준화] 항목, 향후 기능 확장 예정
  - Drive 프리셋에서만 활성화, Local 프리셋은 자동 비활성화

### Fixed (버그 수정)

- **양면 유니폼 그레이딩 버그 4종 해결**
  1. **표/이 스왑 버그** — 표1 자리에 이1이 배치되던 문제 (`isTop` 부등호 통일로 해결)
  2. **색상 대비 반전 버그** — 양면 4분면 색상이 좌우/상하로 뒤집히던 문제 (위 버그 해결에 따라 자동 수복)
  3. **스케일링 부족 버그** — 2XS 등 작은 사이즈에서 스케일이 덜 적용되던 문제 (`ELEMENT_SCALE_EXPONENT` 0.78 → 1.0)
  4. **외측 위 쏠림 버그** — 이름 기반 매칭 시 일부 조각이 위쪽에 몰려 배치되던 문제 (폴백 배치 함수 재사용 구조로 교체)
- **G드라이브 신규 사이즈 SVG가 UI에 반영 안 되는 버그**
  - `mergeDriveScanResult`에서 신규 사이즈 누락 → 기존 치수 보존 + 신규 사이즈 자동 추가 + `SIZE_LIST` 정렬 로직으로 해결

### Changed (기존 동작 변경)

- **스케일링 계수 강화**: `ELEMENT_SCALE_EXPONENT` 0.78 → 1.0
  - 영향: 작은 사이즈(2XS, XS)에서 장식 요소가 더 작아지고, 큰 사이즈에서 더 커짐
  - 이전 버전에서 "작은 사이즈인데 로고가 너무 커 보였다"면 이번에 해결됩니다

### Internal (내부 구조 변경, 일반 사용자 영향 없음)

- 자동 업데이트를 위한 CI/CD 파이프라인 신규 구축
  - `.github/workflows/release.yml`: 태그 푸시 시 자동 빌드/서명/업로드
  - `scripts/bump-version.mjs`: 3파일(package.json / Cargo.toml / tauri.conf.json) 버전 동기화
  - `scripts/sync-bundle-resources.mjs`: `python-engine/*.py`, `illustrator-scripts/*.jsx` 자동 스캔하여 번들 리소스 등록
- Tauri 플러그인 추가
  - `tauri-plugin-updater` (자동 업데이트 감지/다운로드/검증)
  - `tauri-plugin-process` (업데이트 완료 후 앱 자동 재시작)
- `python-engine/svg_normalizer.py` 신규 모듈 (950줄, Phase 1 표준화 엔진)
- `python-engine/order_parser.py`, `python-engine/svg_normalizer.py` 번들 리소스에 등록 (기존 누락분 보충)
