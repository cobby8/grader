# 디자인 AI 레이어 이름 규약 (제안)

> **핵심 철학**: 디자이너가 레이어 이름만 규칙대로 지으면, 코드가 알아서 매칭한다.
>
> 작성일: 2026-04-16 | 대상: grading.jsx (ExtendScript ES3)

---

## 1. 기본 원칙

1. **레이어 이름이 곧 주소다** — 코드는 레이어 이름만 보고 "이 요소가 어느 조각 위에 올라가야 하는지" 판단한다.
2. **구분자는 언더스코어(`_`)** — 공백이나 하이픈 대신 `_`를 쓴다. 파싱이 확실하고 오타가 적다.
3. **한국어 키워드 사용** — 디자이너가 익숙한 한국어를 그대로 쓴다. Illustrator는 한국어 레이어명을 완벽 지원한다.
4. **순서 무관** — 레이어 패널에서 위/아래 순서는 자유. 이름만 규칙대로면 코드가 알아서 찾는다.
5. **규약 미적용 파일 = 기존 방식 자동 폴백** — 이름 규약이 없는 구형 파일은 기존 유클리드 거리 매칭으로 동작한다.

---

## 2. 레이어 이름 체계

### 2-1. 패턴선 / 몸판 레이어 (기존 유지)

| 레이어 이름 | 역할 | 변경 |
|------------|------|------|
| `패턴선` | 모든 패턴 조각의 외곽선 (path) | 없음 |
| `몸판` | 각 조각의 색상 fill (path) | 없음 |

패턴선과 몸판은 기존처럼 1개 레이어에 모든 조각이 들어있어도 된다.
코드가 path의 크기/위치로 자동 분류하는 기존 로직(`classifyBodyPieces`)이 잘 동작하기 때문이다.

### 2-2. 요소 레이어 (신규 규약 — 핵심 변경점)

**현재 문제**: `요소` 레이어 1개에 로고, 번호, 이름표 등이 전부 섞여있다.
코드가 "이 로고는 앞판? 뒷판?" 판별할 때 유클리드 거리(직선 거리)로 추측하는데, 조각 배치가 조금만 달라도 틀린다.

**해결**: 요소 레이어를 조각별로 분리한다.

#### 이름 형식

```
요소_[면]_[조각]
```

| 필드 | 설명 | 허용값 |
|------|------|--------|
| `요소` | 고정 접두사 | `요소` (항상 동일) |
| `면` | 단면이면 생략. 양면이면 `표` 또는 `이` | `표`, `이`, 생략 |
| `조각` | 어느 몸판 조각인지 | `앞`, `뒤`, `좌소매`, `우소매`, `칼라` 등 |

#### 단면 (면 생략)

| 레이어 이름 | 의미 |
|------------|------|
| `요소_앞` | 앞판에 올라가는 모든 요소 |
| `요소_뒤` | 뒷판에 올라가는 모든 요소 |
| `요소_좌소매` | 좌소매에 올라가는 요소 |
| `요소_우소매` | 우소매에 올라가는 요소 |

#### 양면 (면 표기)

| 레이어 이름 | 의미 |
|------------|------|
| `요소_표_앞` | 표면 앞판 요소 |
| `요소_표_뒤` | 표면 뒷판 요소 |
| `요소_이_앞` | 이면 앞판 요소 |
| `요소_이_뒤` | 이면 뒷판 요소 |

### 2-3. 특수 레이어

| 레이어 이름 | 역할 | 규약 |
|------------|------|------|
| `교체용요소` | 백넘버 숫자 등 사이즈별 교체 항목 | 기존 유지 (그대로) |
| `띠_[조각]` | 소매단/밑단 등 띠 요소 (선택) | 예: `띠_앞`, `띠_좌소매` |
| `요소` (기존) | 규약 미적용 시 폴백용 | 이 이름이 있으면 기존 거리 방식 |

## 3. 의류 타입별 예시

### 3-1. 농구 유니폼 단면 (2조각)

```
📁 패턴선
📁 몸판
📁 요소_앞        ← 앞판: 팀로고, 가슴번호, 스폰서
📁 요소_뒤        ← 뒷판: 백넘버, 이름, 뒷면 로고
📁 교체용요소
```

### 3-2. 농구 유니폼 양면 (4조각)

```
📁 패턴선
📁 몸판
📁 요소_표_앞      ← 표면 앞판
📁 요소_표_뒤      ← 표면 뒷판
📁 요소_이_앞      ← 이면 앞판
📁 요소_이_뒤      ← 이면 뒷판
📁 교체용요소
```

### 3-3. 축구 유니폼 (4조각: 앞판+뒤판+좌소매+우소매)

```
📁 패턴선
📁 몸판
📁 요소_앞        ← 앞판: 엠블럼, 가슴번호, 스폰서
📁 요소_뒤        ← 뒷판: 백넘버, 선수명
📁 요소_좌소매     ← 좌소매: 리그 패치
📁 요소_우소매     ← 우소매: 스폰서 패치
📁 교체용요소
```

### 3-4. 하의 (2조각: 앞판+뒤판)

```
📁 패턴선
📁 몸판
📁 요소_앞        ← 앞판: 팀로고
📁 요소_뒤        ← 뒷판: (보통 비어있음)
📁 교체용요소
```

### 3-5. 야구 유니폼 (6조각 이상)

```
📁 패턴선
📁 몸판
📁 요소_앞        ← 앞판 몸통
📁 요소_뒤        ← 뒷판 몸통
📁 요소_좌소매     ← 좌소매
📁 요소_우소매     ← 우소매
📁 요소_칼라      ← 칼라(옷깃) — 해당 시
📁 교체용요소
```

### 3-6. 일반 확장 (N조각)

조각이 추가되면 레이어만 추가하면 된다. 코드 수정 불필요.

```
요소_[새조각이름]         ← 단면
요소_표_[새조각이름]      ← 양면 표면
요소_이_[새조각이름]      ← 양면 이면
```

## 4. 코드가 파싱하는 방법

### 4-1. 레이어 이름 파싱 알고리즘

코드는 모든 레이어를 순회하며 `요소`로 시작하는 레이어를 찾는다.

```
레이어 이름: "요소_표_앞"
     ↓ 언더스코어로 split
["요소", "표", "앞"]
     ↓ 해석
접두사 = "요소"  (요소 레이어 확인)
면    = "표"    (표면)
조각   = "앞"    (앞판)
```

파싱 규칙:

| 토큰 수 | 해석 |
|---------|------|
| 1개: `요소` | 구형 파일. 폴백 모드 (유클리드 거리) |
| 2개: `요소_앞` | 단면. 면 = 없음, 조각 = 두 번째 토큰 |
| 3개: `요소_표_앞` | 양면. 면 = 두 번째, 조각 = 세 번째 토큰 |

### 4-2. 조각 이름 → 몸판 매핑

코드는 조각 이름과 몸판 path를 매칭해야 한다. 방법:

1. **몸판 path를 x좌표 기준 좌→우 정렬** (기존 `classifyBodyPieces` 로직)
2. **조각 이름 → 정렬 순서 매핑 테이블** 사용:

| 조각 이름 | 기본 순서 (좌→우) | 비고 |
|----------|-----------------|------|
| `앞` | 0 (가장 왼쪽) | 대부분의 AI 파일에서 앞판이 왼쪽 |
| `뒤` | 1 | 뒷판은 오른쪽 |
| `좌소매` | 2 | 소매가 있으면 추가 |
| `우소매` | 3 | |
| `칼라` | 4 | |

> 주의: 실제 AI 파일에서 앞/뒤 좌우 위치가 반대인 경우 config에서 순서를 재지정할 수 있도록 설계한다.

### 4-3. 폴백: 규약 미적용 파일

`요소`라는 이름의 레이어만 있고 `요소_*` 패턴이 없으면:
- 기존 유클리드 거리 기반 `assignElementToPiece()` 로직을 그대로 사용
- 로그에 "레이어 이름 규약 미적용 — 거리 기반 폴백" 경고 출력

이렇게 하면 **기존 파일이 깨지지 않는다**.

## 5. 패턴선 / 몸판 레이어 규약

### 5-1. 현재 방식 (유지)

패턴선과 몸판은 1개 레이어 안에 모든 조각의 path가 들어있다.
코드가 path 크기(높이 500pt 기준)로 body/band를 자동 분류하고, x좌표로 좌→우 정렬한다.

**이 방식을 유지하는 이유:**
- 패턴선/몸판은 path 개수가 적고 크기 차이가 명확해서 자동 분류가 안정적
- 디자이너 작업량 최소화 (레이어 분리 불필요)

### 5-2. 선택적 확장: 몸판도 조각별 분리

만약 자동 분류가 불안정한 특수 패턴이 있다면, 몸판도 분리할 수 있다:

```
📁 몸판_앞
📁 몸판_뒤
📁 몸판_좌소매
```

코드는 `몸판_*` 패턴을 발견하면 자동 분류를 건너뛰고 이름 기반으로 매핑한다.
`몸판` 하나만 있으면 기존 자동 분류를 사용한다.

## 6. 마이그레이션 가이드

### 6-1. 기존 파일 변환 방법

기존 `요소` 레이어를 조각별로 분리하는 작업:

1. Illustrator에서 기존 파일을 연다
2. `요소` 레이어를 펼쳐서 내부 아이템을 확인한다
3. 앞판에 해당하는 아이템들을 선택한다
4. 레이어 패널에서 `요소_앞` 레이어를 새로 만든다
5. 선택한 아이템들을 `요소_앞`으로 드래그한다
6. 뒷판, 소매 등도 같은 방식으로 분리한다
7. 기존 `요소` 레이어가 비었으면 삭제한다

> 팁: 기존 `요소` 레이어를 남겨둬도 된다. 비어있으면 코드가 무시한다.

### 6-2. 신규 파일 제작 가이드

새 디자인 파일을 만들 때:

1. `패턴선` 레이어 생성 → 패턴 외곽선 작업
2. `몸판` 레이어 생성 → 조각별 색상 fill
3. 조각별 요소 레이어 생성:
   - 단면: `요소_앞`, `요소_뒤`, (소매 있으면 `요소_좌소매`, `요소_우소매`)
   - 양면: `요소_표_앞`, `요소_표_뒤`, `요소_이_앞`, `요소_이_뒤`
4. 각 요소 레이어에 해당 조각의 요소만 배치
5. `교체용요소` 레이어 생성 (필요시)

## 7. 조각 이름 사전

코드가 인식하는 표준 조각 이름 목록. 이 목록에 없는 이름도 사용 가능하지만,
몸판 매핑 순서를 config에서 별도 지정해야 한다.

| 조각 이름 | 영문 약어 | 설명 |
|----------|----------|------|
| `앞` | front | 앞판 몸통 |
| `뒤` | back | 뒷판 몸통 |
| `좌소매` | sleeve_l | 왼쪽 소매 |
| `우소매` | sleeve_r | 오른쪽 소매 |
| `칼라` | collar | 옷깃 |
| `포켓` | pocket | 주머니 |
| `후드` | hood | 후드 |

면 이름:

| 면 이름 | 설명 |
|--------|------|
| `표` | 표면 (겉면) |
| `이` | 이면 (안면) |

## 8. FAQ

**Q1. 레이어 이름에 공백을 써도 되나요?**
A. 안 됩니다. 구분자는 반드시 언더스코어(`_`)입니다. `요소 앞`은 인식 안 됩니다. `요소_앞`으로 써주세요.

**Q2. 조각에 요소가 없으면 빈 레이어를 만들어야 하나요?**
A. 아닙니다. 요소가 없는 조각은 레이어를 안 만들어도 됩니다. 코드는 있는 레이어만 처리합니다.

**Q3. 기존 파일도 레이어를 바꿔야 하나요?**
A. 당장은 아닙니다. 기존 `요소` 레이어 1개짜리 파일은 기존 거리 기반 방식으로 동작합니다.
다만 그레이딩 결과가 부정확한 파일은 레이어를 분리하면 확실히 해결됩니다.

**Q4. `요소_앞`과 `요소_뒤` 순서가 레이어 패널에서 바뀌어도 괜찮나요?**
A. 네. 코드는 레이어 순서가 아니라 이름으로 판별합니다. 순서는 자유입니다.

**Q5. 양면 파일인데 `요소_앞`으로 쓰면 어떻게 되나요?**
A. 면 구분이 없으면 "단면" 취급됩니다. 양면 파일은 반드시 `요소_표_앞` / `요소_이_앞` 형식을 사용해주세요.

**Q6. 띠(소매단/밑단)에 요소가 있으면 어떻게 하나요?**
A. `띠_앞` 같은 레이어를 만들어 넣으면 됩니다. 또는 해당 조각 레이어(`요소_앞`)에 넣어도 코드가 위치 기반으로 띠 위에 배치합니다.

**Q7. 영어로 이름을 지어도 되나요?**
A. 코드가 한국어 키워드를 기준으로 파싱하므로, 한국어 규약을 따라주세요. 향후 영문 별칭 지원도 고려 중입니다.

**Q8. 하나의 요소가 두 조각에 걸쳐있으면?**
A. 더 많이 걸친 쪽의 조각 레이어에 넣어주세요. 코드는 요소를 해당 조각 기준으로 스케일/배치합니다.

```
📁 패턴선        ← 모든 패턴 조각
📁 몸판          ← 모든 몸판 조각
📁 요소          ← 모든 요소 (뒤섞임) ← 문제!
📁 교체용요소
```
코드가 유클리드 거리로 추측 → 스타일마다 예외 발생

### After (규약 적용)
```
📁 패턴선        ← 모든 패턴 조각 (변경 없음)
📁 몸판          ← 모든 몸판 조각 (변경 없음)
📁 요소_앞       ← 앞판 요소만
📁 요소_뒤       ← 뒷판 요소만
📁 요소_좌소매    ← 좌소매 요소만 (있으면)
📁 교체용요소     ← 변경 없음
```
코드가 이름으로 확정 → 모든 스타일 1개 코드로 대응
